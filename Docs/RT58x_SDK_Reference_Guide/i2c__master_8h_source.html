114 /* Notice:
115  * Because some i2c device can not suppport more than 400KHz,
116  * it means it will waste huge waiting time for block/polling mode during data transfer.
117  * The driver does NOT use polling mode to check I2C transfer finish, it uses interrupt mode.
118  * So the driver is implemented in asynchronous non-block functions for i2c_read/i2c_write transfer,
119  * that is, when i2c_write/i2c_read function return does NOT mean transfer finish.
120  * i2c_write/i2c_read function should give one function parameter ---
121  * "finish callback" hook, this hook will be invoked when the i2c_read/i2c_write
122  * operation finished.
123  *
124  * Please do NOT call i2c_write / i2c_read function in ISR.
